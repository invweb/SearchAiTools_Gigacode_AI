# План: Приложение AI Tools Finder

## Описание проекта
Создание Android-приложения для поиска аналогов AI инструментов с использованием Compose и Material 3.

## Реализуемые экраны

### 1. Экран поиска (Home)
- Поле ввода для поиска AI инструментов
- Фильтрация по категориям (Генерация текста, Генерация изображений, Ассистенты)
- Список с карточками инструментов

### 2. Экран избранного (Favorites)
- Список сохраненных инструментов
- Пустое состояние при отсутствии избранных

### 3. Экран профиля (Profile)
- Информация о приложении
- Описание функционала

### 4. Экран детальной информации
- Открывается при нажатии на инструмент
- Картинка-заглушка для каждого инструмента
- Кнопка добавления в избранное
- Информация об официальном сайте

## Структура данных (Mock-данные)
```kotlin
data class AITool(
    val id: Int,
    val name: String,
    val description: String,
    val url: String,
    val category: Category,
    var isFavorite: Boolean
)
```

**Категории:**
- TEXT_GENERATION (Генерация текста/кода)
- IMAGE_GENERATION (Генерация изображений)
- ASSISTANTS (Чат-боты/ассистенты)

**Примеры инструментов (12 штук):**
- ChatGPT, Claude, Gemini - текстовые ассистенты
- Midjourney, DALL-E, Stable Diffusion - генерация изображений
- GitHub Copilot, Tabnine - генерация кода

## Исправление функционала добавления в избранное

### Проблема:
Раньше на карточке элемента списка не было возможности добавить элемент в избранное напрямую. При нажатии на карточку открывалась только детальная информация.

### Решение:
Добавлена кнопка-переключатель (star icon) в правом верхнем углу каждой карточки элемента списка:
- **Не в избранном** - показывается сердечко с обводкой
- **В избранном** - показывается заполненное сердечко

Теперь пользователь может добавлять/удалять элементы в избранное прямо из списка, а также открывать детальную информацию при нажатии на карточку.

## Технические детали
- Использование Compose для всего UI
- Bottom Navigation для навигации
- State management через mutableStateOf
- Локальные mock-данные без внешних зависимостей
- SearchField вместо SearchBar (из-за ограничений версии Material3)
- Dialog для отображения детальной информации
- Карточка-заглушка с иконкой в качестве изображения
- `stringResource` для всех строковых ресурсов
- Extension функции для работы с ресурсами
